# SP40 — Kafka FDW (query crabka topics as SQL tables) Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a built-in PostgreSQL foreign-data wrapper so any `psql`/JDBC client can `SELECT … FROM <kafka_topic>` against a crabka broker, with values decoded from the Schema Registry into typed SQL rows.

**Architecture:** A new optional crate `kafka_fdw` (behind a `kafka` cargo feature, off by default) depends on crabka's published client + schema-serde crates. The `executor` defines a `ForeignScanner` trait (dependency inversion — executor must not depend on `kafka_fdw`); `kafka_fdw` implements it; the `crabgresql` binary wires the impl into the engine when the feature is on. FDW objects (wrapper/server/user-mapping/foreign-table) are new catalog objects; a `SELECT` on a foreign table branches in `build_table_expr` to the scanner instead of `scan_live`. The scanner uses `crabka-client-core` for a bounded per-partition snapshot read (earliest→high-water-mark-at-start, `read_committed`) and decodes records dynamically against registry schemas.

**Tech Stack:** Rust 2024, tokio (multi-thread runtime), rustls 0.23, `crabka-client-core` / `crabka-client-admin` / `crabka-schema-serde` (0.3.x, crates.io), `apache-avro` / `serde_json` / `prost-reflect` for dynamic decode, `pgparser` / `catalog` / `kv` / `pgtypes` / `executor` / `cluster` (crabgresql workspace).

## Global Constraints

- **Edition 2024; MSRV per workspace; `unsafe_code = "forbid"`; `clippy::unwrap_used = "warn"`** — use `expect("reason")` everywhere, tests included (CI runs `-D warnings`).
- **Zero-C is preserved for the DEFAULT build.** All new C-pulling deps are **optional**, gated behind the `kafka` feature. `scripts/check-no-native.sh` and `cargo-deny` must stay green on the default feature set. The `kafka` CI lane is the only place C is permitted.
- **No `sleep()` in tests.** Wait on conditions (broker-ready, applied state) bounded by a timeout. Add instrumentation rather than sleeping.
- **`pgparser` is a mutation-clean baseline crate** (zero missed mutants). Every new parser branch needs a unit test that kills its mutant. Run `cargo mutants -p pgparser` after Task 2.
- **`pgtypes` is also a mutation-clean baseline crate** — same discipline for Task 4.
- **Conventional commits**: `feat:` / `fix:` / `test:` / `chore:`. End every commit body with `Co-Authored-By: Claude Opus 4.8 <noreply@anthropic.com>`.
- **`pgtypes` has no `Bytea`, `Jsonb`, or `Float4` today.** Task 4 adds `Bytea` (needed for `_key` and the raw-value fallback). `_headers` and nested Avro/JSON (`record`/`array`/`map`) map to **`text`** (JSON-serialized) in this slice; a real `Jsonb` type is deferred. Avro `float` decodes to `float8` (no `Float4`).
- **`exec.rs` is the serialization point.** Tasks 6, 13, 14 all edit `crates/executor/src/exec.rs`; they must run sequentially (never two subagents on `exec.rs` at once).
- **Envelope columns** are always the first five columns of every Kafka foreign table, in this fixed order and type: `_partition int4`, `_offset int8`, `_timestamp timestamptz`, `_key bytea`, `_headers text`. Catalog `create_foreign_table` / `IMPORT` prepend them to the declared/imported value columns.

```rust
// crates/executor/src/foreign.rs — dependency-inversion seam (Task 6)
#[derive(Debug, Clone, Default)]
pub struct ScanBounds {
    pub partitions: Option<Vec<i32>>,   // None = all partitions
    pub offset_lo: Option<i64>,         // inclusive
    pub offset_hi: Option<i64>,         // exclusive
    pub ts_lo_ms: Option<i64>,          // inclusive, epoch ms
    pub ts_hi_ms: Option<i64>,          // exclusive, epoch ms
    pub limit: Option<usize>,
}

/// Implemented by `kafka_fdw::KafkaFdw`; registered on the engine by the binary.
/// Sync signature: the impl drives async Kafka I/O via
/// `tokio::task::block_in_place(|| Handle::current().block_on(fut))`
/// (requires the multi-thread runtime crabgresql already uses).
pub trait ForeignScanner: Send + Sync {
    fn scan(
        &self,
        table: &catalog::Table,                 // table.foreign = Some(meta)
        server: &catalog::ForeignServer,
        mapping: Option<&catalog::UserMapping>,
        bounds: &ScanBounds,
        ctx: &crate::clock::EvalCtx,
    ) -> Result<Vec<Vec<pgtypes::Datum>>, ExecError>; // rows aligned to table.columns (envelope first)

    /// For IMPORT FOREIGN SCHEMA: list (table_name, value_columns) for each subject.
    fn import_schema(
        &self,
        server: &catalog::ForeignServer,
        mapping: Option<&catalog::UserMapping>,
        selector: &ImportFilter,
    ) -> Result<Vec<(String, Vec<catalog::Column>)>, ExecError>;
}
pub enum ImportFilter { All, LimitTo(Vec<String>), Except(Vec<String>) }
```

```rust
fn parse_options(&mut self) -> Result<crate::ast::OptionList, ParseError> {
    // OPTIONS ( ident 'string' [, …] )  — empty list if OPTIONS absent
    if !self.eat_keyword(Keyword::Options) { return Ok(vec![]); }
    self.expect(&Token::LParen)?;
    let mut opts = Vec::new();
    loop {
        let k = self.expect_ident()?;
        let v = self.expect_string_lit()?;   // add expect_string_lit mirroring expect_ident
        opts.push((k, v));
        if self.eat_comma() { continue; }
        break;
    }
    self.expect(&Token::RParen)?;
    Ok(opts)
}
```

1. **Provider coexistence (Task 5) may require an upstream crabka change.** Gated first; the fallback (inject a provider into `ClientSecurity`) is small and additive. Do not start Task 8 until Task 5 is green.

4. **In-process crabka broker bring-up (Task 16)** is the least-certain piece; the child-process fallback is documented. If the broker lib API is awkward, prefer the binary-spawn path that mirrors crabgresql's existing `ProcNode` harness.
