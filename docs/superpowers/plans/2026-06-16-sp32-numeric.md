# SP32 plan — `numeric` / `decimal` (arbitrary-precision exact decimal) (SQL breadth wave 6)

Design: `docs/superpowers/specs/2026-06-16-crabgresql-sp32-numeric-design.md`.
Branch: `claude/eager-volta-dd4izk`.

Sixth breadth slice. Adds PostgreSQL `numeric`/`decimal` (OID 1700) — arbitrary-precision
exact decimal with a tracked display scale and the `numeric(p,s)` modifier — retiring the
SP27/SP30/SP31 "no numeric" deviations: bare decimal literals become scale-faithful
`numeric`, `avg(int)` becomes exact `numeric`, and exact numeric arithmetic exists. Backed by
the `bigdecimal` crate (one new dependency, pure Rust). Pure-data / single-range → **no
Stateright model** (see spec). Proven by unit + integration + the differential conformance
oracle (validated locally against a real PostgreSQL 16, diffed against PG 18 in CI).

## Task 1 — Types (`pgtypes`)
- [x] `numeric.rs` (new): `Typmod`; `canonical` (dscale ≥ 0); `parse`; `to_text` (hand-written
  `numeric_out`, plain decimal); `binary` (`numeric_send` NBASE); `add`/`sub`/`mul`;
  `div`/`rem` (+ `select_div_scale`, div-by-zero 22012); `abs`; `to_i32`/`to_i64` (half-away +
  range 22003); `from_i64`/`from_f64` (shortest text)/`to_f64`; `apply_typmod` (round +
  precision overflow 22003). `lib.rs`: `pub mod numeric`.
- [x] `datum.rs`: `oids::NUMERIC`; `ColumnType::Numeric(Option<Typmod>)` + `is_numeric()`;
  `from_sql_name` (`numeric`/`decimal`); oid/name/typlen; `Datum::Numeric(BigDecimal)` with
  value-based `PartialEq`/`Hash` (scale-ignoring grouping) + `column_type`.
- [x] `encoding.rs`: numeric text (`numeric::to_text`) + binary (`numeric::binary`).
- [x] `ops.rs`: numeric tier (int < numeric < float8) in `add`/`sub`/`mul`/`div`/`rem`/`compare`
  (`as_numeric`, `is_numeric`; `as_f64` handles numeric for the float-wins case).
- [x] `cast.rs`: numeric family in `cast_allowed` + `cast` (int/float/text/numeric → numeric
  with typmod; numeric → int/float/text; no numeric↔bool). `error.rs`: reused 42846/22P02/22003.
- [x] Unit tests (mutation-baseline crate → exhaustive): div scale battery, rounding, typmod,
  NBASE bytes, grouping eq, the cast matrix.

## Task 2 — Storage (`kv`, `catalog`)
- [x] `kv::rowenc`: tag `NUMERIC = 6` (canonical decimal text, round-trips scale); roundtrip +
  `arb_datum`.
- [x] `catalog::serde`: tag `NUMERIC = 5` with an inline typmod payload (`write_type`/`read_type`);
  roundtrip incl. `numeric(10,2)` and unconstrained.

## Task 3 — Parser/AST (`pgparser`)
- [x] `ast.rs`: `Expr::FloatLiteral` → `Expr::NumericLiteral` (bare decimals are numeric).
- [x] `parser.rs`: `NumericLiteral` prefix arm; `parse_type_name` parses `numeric(p[,s])`
  (`parse_numeric_typmod`, `expect_u16`); cast target accepts the modifier.
- [x] Unit tests: numeric literals, `numeric(p,s)` columns + cast target.

## Task 4 — Executor (`eval`, `agg`, `func`, `exec`)
- [x] `eval.rs`: `NumericLiteral` in `eval`/`infer_type`; numeric tier in
  `numeric_result_type`/`unify_types`.
- [x] `agg.rs`: `SumN`/`AvgN` accumulators (exact; AVG via `select_div_scale`), numeric
  `MIN`/`MAX`; `func_result_type`/`spec_of` accept numeric; `finish` → `Result`;
  `eval_grouped` `NumericLiteral`.
- [x] `func.rs`: `abs`/`mod` extended to numeric (`require_int_or_numeric`; PG has no float mod).
- [x] `exec.rs`: `coerce` rounds/overflows numeric-family → numeric column; numeric → int/float8
  assignment.
- [x] Adjust SP30 float tests that assumed bare-literal-float8 (`eval`, `agg`, `func`,
  `tests/floating_point.rs`) to use `::float8` / numeric expectations.
